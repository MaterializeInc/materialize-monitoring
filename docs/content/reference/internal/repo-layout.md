title: "Repo Layout"
weight: 10
---


* `materialize-monitoring/`
  * `Cargo.toml`: Top-level Rust workspace
  * `Cargo.lock`
  * `bin/`: Bash CI/dev entrypoints (flat; no subdirectories)
    * `ci-lint`: Orchestrates linters; calls Rust binaries where logic is needed
    * `ci-build`: Orchestrates artifact generation
    * `ci-test`: Orchestrates the test layers
    * `ci-publish`
  * `tools/`: Ancillary projects with their own ecosystems, kept out of bin/
    * `kind-assets/`: YAML assets used by the kind/ArgoCD/FluxCD CI matrix
  * `.github/`
    * `workflows/`: GitHub Actions workflow definitions; thin wrappers over bin/ scripts
  * `.pre-commit-config.yaml`: Comprehensive contributor-experience hooks (see Pre-commit section)
  * `packages/`: Hand-authored, contributor-facing inputs
    * `grafana-dashboards/`: Grafana SDK-sourced dashboards and panels
    * `datadog-dashboards/`: Datadog SDK-sourced dashboards
    * `mz-monitoring/`: monitoring definitions (alerts, rules, queries, pipelines)
    * `py-mzmon-lib/`: Python helpers for dashboard generation; imported by other packages but not consumed by customers
    * `mz-monitoring-build/`: Build orchestration: reads YAML, drives Jsonnet, emits artifacts
    * `mz-monitoring-check/`: Schema validation, label-contract conformance, cross-artifact consistency
    * `mzmon-lib/`: Rust library for helpers in mz-monitoring-build and mz-monitoring-check; not consumed by customers
  * `charts/`
    * `materialize-monitoring/`: Umbrella chart
      * `Chart.yaml`
      * `Chart.lock`: Pins subchart versions
      * `values.yaml`: Profile-driven defaults; parametrized comments drive README generation
      * `values.schema.json`
      * `README.md`: Generated by helm-readme-sync; CI-verified against values.yaml
      * `charts/`: Vendored subchart tarballs (see open question)
        * `alloy-$version.tgz`
        * `loki-$version.tgz`
        * `thanos-$version.tgz`
        * `alertmanager-$version.tgz`
        * `grafana-$version.tgz`
        * `kube-state-metrics-$version.tgz`
        * `metrics-server-$version.tgz`
        * `prometheus-crds-$version.tgz`
      * `pre-rendered/`: Generated artifacts the chart loads via {{ .Files.Get }}; never hand-edited
        * `dashboards/`
          * `grafana/`: JSON output from packages/grafana-dashboards
          * `datadog/`: JSON output from packages/datadog-dashboards
        * `rules/`
          * `prometheus/`: Expanded YAML from mz-monitoring/rules/
          * `loki/`
          * `thanos/`
        * `README.md`: "Generated; edit sources/ and regenerate" pointer
      * `templates/`: Provided resources (e.g., shared ConfigMaps)
        * `alerts/`
        * `dashboards/`: Grafana-operator importable content
        * `pipelines/`: Metric/logging pipelines for e.g., alloy
        * `scrapers/`: Service Monitors / Scrape Configs for gaps in first-class supports (like Materialize)
  * `images/`
    * `alloy-distroless/`: Distroless Alloy image used by both runtime and pre-install validation hook
  * `legacy/`
    * `sql_exporter/`: Preserved from MaterializeIncLabs
  * `docs/`
    * `hugo.yaml`: Site config; theme/shortcode conventions match the materialize repo
    * `content/`: Authored Markdown
      * `_index.md`: Landing page
      * `profiles/`: Profile catalog (one page per profile)
      * `alerting/`
      * `dashboards/`
      * `getting-started/`
      * `logs-and-events/`
      * `metrics/`
      * `operating/`
      * `reference/`
        * `internal/`: Reference content for repo contributors and AI agents; not public-facing documentation
          * `dashboard/`: Stable conventions for dashboards-as-code project
          * `releasing.md`: Releasing process documentation (in progress)
          * `contract/`: Public-facing rendering of the contract docs
        * `helm/`
        * `architecture.md`
        * `changelog.md`
    * `public/`: Generated site output (not checked in)
    * `static/`
    * `layouts/`: Hugo layout overrides
