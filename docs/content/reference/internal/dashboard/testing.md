# Testing

Dashboard code is tested with `pytest`. Unit tests are recommended to be placed next to their code with the `_test.py` suffix.

For broader code-quality tooling (`ruff`, `pyright`, configured linter rules), see [Generating and Pushing Dashboards > Code quality]({{< relref "generating.md#code-quality" >}}).
